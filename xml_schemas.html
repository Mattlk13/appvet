AppVetProperties

The AppVet Properties XML Schema defines the configurable parameters of AppVet. This schema is defined by the file AppVetProperties.xsd. An example instance of this schema is shown in AppVetProperties.xml. The configurable instance of this schema is found in $APPVET_FILES_HOME/conf/AppVetProperties.xml .

The following are the configurable properties defined in the AppVet Properties XML schema.

Admin

The Admin property allows for the specification of the AppVet administrator username and password . Setting Admin is required to launch AppVet for the first time.

Host

The Host parameter specifies the hostname (or IP address) of the AppVet server that client should use. Due to various issues with DNS servers in obtaining a correct hostname, AppVet requires the manual specification of this parameter. In addition, the port and SSL parameters for the server must also be specified.

Logging

The Logging parameter specifies the logging configuration of AppVet. The Level parameter specifies the types of logging statements to be output including DEBUG , INFO , WARN , and ERROR statements. In addition, the ToConsole parameter indicates whether logging should be output to the console, and is typically used for debugging during AppVet development.

UseSSO

The UseSSO parameter specifies whether AppVet should use Single Sign-On (SSO) for all transactions. If SSO is used, a separate URL that supports SSO will be used for AppVet to provide authentication. For more information, see AppVet Single Sign-On (SSO).

Sessions

The Sessions parameter specifies sessions-related configurations including the Timeout that specifies the maximum amount of idle time by a user (in milliseconds) and the GetUpdatesDelay that specifies the delay (in milliseconds) that AppVet will check for app-related updates (e.g., app processing status updates).

Database

The Database parameter specifies database-related settings including the URL of the database as well as the username and password to use to authenticate with the database.

ToolServices

The ToolServices parameter specifies tool-related settings concerned with delays and timeouts. The PollingInterval specifies the delay (in milliseconds) that AppVet waits before checking for the next app to process. The StaggarInterval specifies the delay (in milliseconds) that AppVet waits before submitting an app to a testing service. The ConnectionTimeout specifies the time to wait (in milliseconds) for a connection to a testing service before timing out. The SocketTimeout specifies the time to wait (in milliseconds) for incoming data from a testing service before timing out.

Apps

The Apps parameter specifies app-related settings. Currently, only the KeepApps property is used to indicate whether AppVet should keep uploaded apps on the system. Note that keeping uploaded apps on the system can quickly use up disk storage on the system. By default, KeepApps is set to false .

ToolAdapter

The Tool Adapter XML Schema defines an online testing service to be integrated with AppVet. This schema is defined by the file ToolAdapter.xsd. An example instance of this schema is shown in ToolAdapter.xml.

Integration of a testing service adapter involves adding a vendor-supplied ToolAdapter.xml file into $APPVET_FILES_HOME/conf/tool_adapters by the AppVet administrator. In general, testing service adapters will be developed and provided by the vendors of the specific tool services and should not be modified by the AppVet administrator.

ToolProperties

The Tool Properties XML Schema defines the configurable parameters of a testing service. This schema is defined by the file ToolProperties.xsd. An example instance of this schema is shown in ToolProperties.xml.

In general, tool services are developed and provided by vendors. Configurable instances of this schema should be placed in $<TOOL_FILES_HOME>/conf/ToolProperties.xml on the . vendor's server.

Starting AppVet

To start AppVet, start the Tomcat service and open a browser to your AppVet URL https://<host>:<port>/appvet where <host>:<port> is the hostname or IP address and port number of your server. The AppVet login screen should be visible as shown in Figure 1.

Using AppVet

After logging into AppVet, the AppVet app management interface is displayed as shown in Figure 2.





Figure 2. AppVet UI (risk scores redacted for privacy).



The AppVet app management interface comprises two main panels: an apps list panel on the left and an app information panel on the right. The apps list panel displays apps that have been uploaded to the system while the app information panel displays information about the selected app (highlighted in blue in the apps list panel). The apps list panel displays general information about uploaded apps including their AppVet-generated app ID, app name, current status and risk score, user (app owner), and the date/time when the app was uploaded to the system. The app info panel contains information about a selected app including:

  • App name and icon
  • Version number
  • Registration and app pre-processing statuses
  • Tool reports
  • Tool status and risk scores

Operation Buttons

The apps list panel contains operation buttons used to manage apps and their related reports. Table 1 describes the functions of AppVet operation buttons.

Table 1. Operation buttons.

Name Description
Upload App Upload an app file.
Upload Report Upload a report for an analysis (test) tool or overview report for the selected app.
View Log View the processing log for the selected app.
Delete App Deletes the selected app.
Download Reports Download all reports for the selected app as a .zip file.
Download app Download the selected app.


Tool and Report Statuses

In the app information panel, each tool or report is associated with a processing status or risk score as shown in Figure 4.



Figure 4. App Tools Status.



A tool or report status indicates the current processing status of, or the risk score generated by, a tool or analyst. AppVet requires that risk scores be mapped to one of three risk categories: LOW, MODERATE, or HIGH. Table 2 describes these risk score categories and AppVet tool statuses.



Table 2. Tool statuses and risk scores.

Type Tool Status Description
Processing Status N/A No status information is available for the tool.
PENDING App is waiting to be submitted to the specified tool.
SUBMITTED App has been submitted to the specified tool.
COMPLETED Pre-processing of the app has been completed.
AVAILABLE A report is available.
ERROR The tool could not analyze the app.
Risk score LOW The tool (or analyst) has designated the app as low-risk.
MODERATE The tool (or analyst) has designated the app as moderate-risk.
HIGH The tool (or analyst) has designated the app as high-risk.



Note that a risk score is only displayed after the tool has successfully completed processing. Further note that risk scores generated by a tool can be overridden if needed. This feature is used to mitigate false positive analysis results.

App Status

In the apps list panel, each app is associated with a status or risk score as shown in Figure 5.




Figure 5. App status.




An app status indicates the current status or risk score of an app which, in turn, is based on the statuses and risk scores of the tools. Table 3 describes the AppVet app statuses and risk scores.


Table 5-3. App status and risk scores.



Type App Status Description
Processing Status REGISTERING App is being registered by AppVet.
PENDING App is waiting to be analyzed.
PROCESSING App is being analyzed by one or more tools.
ERROR One or more tools could not analyze the app.
Risk score LOW All tools designate the app as low-risk.
MODERATE At least one tool designates te app as moderate-risk, but no tool designates the app as high-risk.
HIGH At least one tool designates the app as high-risk.



Note that if a risk score is displayed, processing for the app has completed.

Note that the decision to approve or reject an app is dependent upon the policies and security requirements of the organization that owns and operates the instance of AppVet. In addition, overall risk scores generated by AppVet are based solely on the risk scores provided by the tools used.

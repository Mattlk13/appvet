Setting Global Environment Variables

  • Step 1: Edit /etc/profile
  • Step 2: Edit Sudoers
AppVet depends on the proper setting of global environment variables. The following describes how to create a global environment variables under CentOS/RedHat. For the following examples, we create a global environment variable $MY_PROGRAM_HOME and set it to the directory /home/jdoe/myprogram.

Step 1. Edit /etc/profile

Edit /etc/profile using sudo (e.g., sudo vi /etc/profile) by adding the following: 

		
	#Global Environment Variables
	MY_PROGRAM_HOME=/home/jdoe/myprogram
	PATH=$PATH:$MY_PROGRAM_HOME
	export MY_PROGRAM_HOME
	export PATH

Next, reload /etc/profile by running:

# source /etc/profile


in all open terminals. To verify that $MY_PROGRAM_HOME has been added to your $PATH variable, run:

# printenv MY_PROGRAM_HOME


Step 2. Edit Sudoers

To allow $MY_PROGRAM_HOME to be accessed when using sudo , open a terminal and run:

# sudo visudo


From visudo , add the following line:

Defaults env_keep += "MY_PROGRAM_HOME"


This will force sudo to preserve and use the $MY_PROGRAM_HOME environment variable as defined in /etc/profile (since sudo typically ignores these environment variables). To verify that $MY_PROGRAM_HOME is now accessible to sudo , run:

# sudo printenv MY_PROGRAM_HOME

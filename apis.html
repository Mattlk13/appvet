AppVet API

AUTHENTICATE

The AUTHENTICATE service accepts a username and password for authenticating a client with AppVet and returns an AppVet session ID. This session ID may then be used by the client for further interaction with AppVet until the session expires. The AUTHENTICATE HTTP Request API is shown in Table 1a.




Table 1a. AUTHENTICATE request.

Entity Name and/or Value Description
Method GET HTTP Request method.
Parameter command = AUTHENTICATE AppVet command.
Parameter username = <username> AppVet username.
Parameter password = <password> AppVet password.



The HTTP Response for an AUTHENTICATE HTTP Request is shown in Table 1b.


Table 1b. HTTP response.

Entity Name and/or Value Description
Status Code <statuscode> HTTP status code.
Payload <sessionid> AppVet session ID.

SUBMIT_REPORT

The SUBMIT_REPORT service submits (or overrides an existing) tool report for the specified app. This service is used by asynchronous and push services to submit reports as well as analysts for overriding existing tool reports. The SUBMIT_REPORT HTTP Request API is shown in Table 9a.


Table 8-9a. SUBMIT_REPORT request.



Entity Name and/or Value Description
Method POST HTTP Request method.
Parameter command = SUBMIT_REPORT AppVet command.
Parameter appid = <appid> AppVet app ID.
Parameter username = <username> AppVet username.
Parameter password = <password> AppVet password.
Parameter toolrisk = <toolrisk> The risk assessment PASS, WARNING, or FAIL. If the service could not process the app, then risk should have a value of ERROR.
Parameter file = <reportfile> Report file.



The HTTP Response for an SUBMIT_REPORT HTTP Request is shown in Table 9b.


Table 9b. HTTP response.



Entity Name and/or Value Description
Status Code <statuscode> HTTP status code.



Tool APIs

Although different online tools with have different APIs, AppVet requires that at the very least, the online tool accept an app ID and an app file. The app ID must be returned with a report and risk assessment by the online tool in order for AppVet to associate the report/risk assessment with the proper app. Table 10 shows the minimum requirements for a tool API.

Entity Name and/or Value Description
Parameter appid = <appid> App ID.
Parameter file = <appfile> App file.


When an online tool sends a report and risk assessment back to AppVet, it does so through the AppVet SUBMIT_REPORT API.

Dependencies

The AppVet binary release requires the installation of platform dependencies. See instructions for installing AppVet dependencies.

Download

After AppVet dependencies have been installed, download the AppVet binary release appvet_2.x_release.tar.gz file. After downloading the AppVet binary release file, extract the compressed file onto your local machine.

Installing AppVet

The AppVet binary release contains the following directories:
  • appvet.war: The AppVet web application.
  • appvet.images: AppVet images.
  • appvet.files: AppVet files including configuration, log, and app files.
To install AppVet, copy the appvet.war file and appvet.images directory to your $CATALINA_HOME/webapps directory. Next, copy the appvet.files directory to your local machine.

Environment Variable

To ensure proper operation of AppVet, the $APPVET_FILES_HOME environment variable must be set to the installation path of your appvet.files directory. For examples, if you copied appvet.files to $HOME/jdoe , then:

$APPVET_FILES_HOME=/home/jdoe/appvet.files

Using sudo , edit /etc/profile by adding the following:

#Global Environment Variables
APPVET_FILES_HOME=/your/path/to/appvet.files
PATH=$PATH:$APPVET_FILES_HOME
export APPVET_FILES_HOME
export PATH

Next, reload /etc/profile by running:

# source /etc/profile

in all open terminals. To verify that $APPVET_FILES_HOME has been added to your $PATH variable, run:

# printenv APPVET_FILES_HOME

Next, to allow $APPVET_FILES_HOME to be accessed when using sudo , open a terminal and run:

# sudo visudo

From visudo , add the following line:

Defaults env_keep += "APPVET_FILES_HOME"

This will force sudo to preserve and use the $APPVET_FILES_HOME environment variable as defined in /etc/profile (since sudo typically ignores these environment variables). To verify that $APPVET_FILES_HOME is now accessible to sudo , run:

# sudo printenv APPVET_FILES_HOME

AppVet Database

To use AppVet, the AppVet database must be created and configured in MySQL.

Create Database

To create the appvet database, log into MySQL by opening a terminal and running:

# mysql -u root -p<password>

Next, at the mysql prompt, enter:

mysql> create database appvet;

Note that all MySQL statements must end with a semicolon. Next, type exit to exit MySQL.

To create the tables for the appvet database, open a terminal and navigate to $APPVET_FILES_HOME/conf/mysql then run:

# mysql -u root -p<password> appvet < appvet_init_db.sql

Note that there will be no notification that the appvet tables were loaded successfully. To verify the loading of the appvet tables, log back into MySQL and enter:

mysql> use appvet;

mysql> show tables;

The tables in the appvet database should include:
  • androidtoolstatus
  • apps
  • iostoolstatus
  • sessions
  • users
See AppVet database schema for more information about these tables.

Configuring AppVet involves modifying the AppVet properties XML file located in $APPVET_FILES_HOME/conf/AppVetProperties.xml (See Example AppVet properties file). The AppVet properties XML file conforms to the AppVet Properties XML Schema.

The following describes the user-configurable AppVet properties.

Admin

The Admin property allows for the specification of the AppVet administrator username and password . Setting Admin is required to launch AppVet for the first time.

Host

The Host property specifies the hostname (or IP address) of the AppVet server that client should use. Due to various issues with DNS servers in obtaining a correct hostname, AppVet requires the manual specification of this property. In addition, the port and SSL properties for the server must also be specified. Note that the port and SSL properties must match your Tomcat configuration.

Logging

The Logging property specifies the logging configuration of AppVet. The Level property specifies the types of logging statements to be output including DEBUG , INFO , WARN , and ERROR statements. In addition, the ToConsole property indicates whether logging should be output to the console, and is typically used for debugging during AppVet development. For production use, ToConsole should be set to false.

UseSSO

The UseSSO property specifies whether AppVet should use Single Sign-On (SSO) for all transactions. If SSO is used, a separate URL that supports SSO will be used for AppVet to provide authentication. For more information, see AppVet Single Sign-On (SSO).

Sessions

The Sessions property specifies sessions-related configurations including the Timeout that specifies the maximum amount of idle time by a user (in milliseconds) and the GetUpdatesDelay that specifies the delay (in milliseconds) that AppVet will check for app-related updates (e.g., app processing status updates).

Database

The Database property specifies the AppVet database-related settings including the URL of the database as well as the username and password used to authenticate with the AppVet database.

ToolServices

The ToolServices property specifies online tool-related settings concerned with delays and timeouts. The PollingInterval specifies the delay (in milliseconds) that AppVet waits before checking for the next app to process. The StaggarInterval specifies the delay (in milliseconds) that AppVet waits before submitting an app to a tool. The ConnectionTimeout specifies the time to wait (in milliseconds) for a connection to a tool before timing out. The SocketTimeout specifies the time to wait (in milliseconds) for incoming data from a tool before timing out.

Apps

The Apps property specifies app-related settings. Currently, only the KeepApps property is used to indicate whether AppVet should keep uploaded apps on the system. Note that keeping uploaded apps on the system can quickly use up disk storage on the system. By default, KeepApps is set to false .

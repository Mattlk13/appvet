Configuring AppVet SSO involves modifying the AppVet SSO properties XML file located in $APPVET_SSO_FILES_HOME/conf/AppVetSSOProperties.xml (See Example AppVet SSO properties file). The AppVet SSO properties XML file conforms to the AppVet SSO Properties XML Schema.

The following describes the user-configurable AppVet SSO properties.

Host

The Host property specifies the hostname (or IP address) of your AppVet server (not your AppVet SSO server). Due to various issues with DNS servers in obtaining a correct hostname, AppVet requires the manual specification of this property. In addition, the port and SSL properties for the server must also be specified. Note that the port and SSL properties must match your Tomcat configuration.

Logging

The Logging property specifies the logging configuration of AppVet SSO. The Level property specifies the types of logging statements to be output including DEBUG , INFO , WARN , and ERROR statements. In addition, the ToConsole property indicates whether logging should be output to the console, and is typically used for debugging during AppVet development. For production use, ToConsole should be set to false.

UserInfo

The UserInfo property specifies the single sign-on parameter names sent by your organization's authentication system to AppVet SSO for (AppVet) authentication. At the very least, UserNameParamName must be set to a parameter name that is sent from an authorization system. Note that parameter names and their corresponding values should be sent to AppVet SSO via HTTP Request headers.

Use JIRA

The UseJira property specifies whether or not JIRA should be used to acquire some or all user information. The default value should be set to false .

JIRA

The Jira property specifies a number of properties of the Jira sever used to acquire user information. For more information, consult the JIRA API.

Database

The Database property specifies the AppVet database-related settings including the URL of the database as well as the username and password used to authenticate with the AppVet database. Note that the values of these properties should be identical to those defined in your $APPVET_FILES_HOME/conf/AppVetProperties.xml file.

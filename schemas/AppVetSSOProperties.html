The AppVet SSO Properties XML Schema defines the configurable parameters of AppVet SSO. This schema is defined by the file AppVetSSOProperties.xsd. An example instance of this schema is shown in AppVetSSOProperties.xml. The configurable instance of this schema is found in $APPVET_SSO_FILES_HOME/conf/AppVetSSOProperties.xml .

The following are the configurable properties defined in the AppVet SSO Properties XML schema.

Host

The Host parameter specifies the hostname (or IP address) of the actual AppVet server (not the AppVet SSO server). Due to various issues with DNS servers in obtaining a correct hostname, AppVet requires the manual specification of this parameter. In addition, the port and SSL parameters for the server must also be specified.

Logging

The Logging parameter specifies the logging configuration of AppVet SSO. The Level parameter specifies the types of logging statements to be output including DEBUG , INFO , WARN , and ERROR statements. In addition, the ToConsole parameter indicates whether logging should be output to the console, and is typically used for debugging during AppVet development.

UserInfo

The UserInfo parameter specifies the single sign-on parameters sent by an authentication system of the organization to AppVet SSO for authentication. At the very least, UserNameParamName must be set to a parameter that is sent from an authorization system.

Use JIRA

The UseJira parameter specifies whether or not JIRA should be used to acquire some or all user information. The default value should be set to false .

JIRA

The Jira parameter specifies a number of properties of the Jira sever used to acquire user information. For more information, consult the JIRA API.

Database

The Database parameter specifies database-related settings including the URL of the database as well as the username and password to use to authenticate with the database.
